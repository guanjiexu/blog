Java高级工程师

Veröffentlicht am 2019-03-29

Java高级工程师中，面试中的常考点，以及工作所需技能。

技能等级：

  • 了解：知道基本的概念，并能快速搭建一些脚手架
  • 熟悉：能够快速使用该工具，清楚其中的常见坑点
  • 精通：熟悉其源码，能够进行二次开发和优化

工作必备

1. 架构方式（微服务）

针对互联网应用，性能要求不高。

技术选型：

  1. 服务间通信
    • RPC，可选用GRPC
    • 注册中心，可参考Eureka，Dubbo
    • 熔断
    • 动态伸缩（可选，参考K8S）
    • 限流
    • 网关：用来做权限管理，路由分配
    • 配置中心：推荐Apollo

  2. 缓存

    • 采用Redis-Cluster

  3. 分布式锁

    • 采用Redlock或zookeeper

  4. 数据落地

    • PGSQL/MySQL配合MyBatis
    • ShardingJDBC（大的数据分片）
    • HBASE（超大规模的简单数据的存储和查询）
  5. 消息队列
    • RabbitMQ/RocketMQ：耗时业务的调用，消峰
    • Kafka：用于简单的日志推送
  6. 分布式事务
    • 由于微服务拆分，分布式事务很重要
  7. 日志与统计

面试常见问题大纲

  1. 理论基础

    • 算法

      • 了解：B/B+树，堆排序，快速排序，跳表，HashMap，堆，队列，一致性Hash，二分查找，红黑树

        QA

        1. B/B+树的区别
        2. 为什么数据库系统不选择红黑树，而选择B/B+树
        3. 写一个堆排序
        4. 写一个快速排序
        5. Redis中跳表的实现
        6. HashMap中碰撞了怎么办
        7. Hash算法，为什么Map容量设计为2的n次方
        8. 解释一下一致性Hash，虚拟节点，一个节点当掉，会产生多少数据迁移

    • 操作系统

      • 了解：进程/线程/协程，虚拟内存，TCP协议，中断，I/O，CPU缓存

        QA

        1. Linux中进程和线程有什么区别
        2. 解释下vfork,fork的流程
        3. Linux中进程的调度方式
        4. 解释下页表，为什么使用页表，fork后进程的页表发生了什么变化
        5. 协程和线程的性能差在什么地方（上下文切换，重载页表项，刷新页表缓存，寄存器重新加载内存，线程内存分配，线程有分时调度，协程不强制）
        6. TCP为什么是3次握手不是2次，为什么是4次挥手，TIME_WAIT为什么是2MSL，解释下nigle算法，快速重传，快速恢复，解释下backlog，如果取消TIME_WAIT（两边同时关闭）
        7. CPU缓存一致性协议，内存屏障，命中率对性能的影响，缓存着色，缓存行

    • 分布式

      • 了解：2PC/3PC，Raft/Paxos，负载均衡的算法与实现（session sticky），熔断，RPC
      • 了解：zookeeper

  2. Java基础

    • 集合：熟悉，着重考察HashMap/HashTable/ConcurrentHashMap（如何扩容，线程安全，发生碰撞了怎么办）
    • NIO：了解，Select/Epoll/Poll区别，动手写NIO处理网络请求
    • 线程池：熟悉，实现原理，几种产生方法
    • JVM：了解，内存区域，类加载器（双亲委派），回收算法，垃圾回收器（CMS/G1）
    • 反射
    • 多线程：violate，线程池，ThreadLocal(内存泄漏，弱引用)，AQS，CyclicBarrir/CountDownLatch，ReentrantLock

  3. SQL

    • 事务（熟悉）：隔离级别，MVCC
    • 锁（了解）：for update，lock
    • 索引（熟悉）：分布式自增，聚集索引，最左原则，Explain
    • MyBatis（熟悉）：二级索引，一级索引，如何join
    • 分库分表（了解）
    • 高可用（了解）：读写分离，双主（keepalive）

  4. Redis

    • 熟悉：数据结构及实现，淘汰策略，AOF/RDB，事务
    • 了解：哨兵模式，集群模式
    • 熟悉：用作缓存，防止雪崩，预热
    • 了解：Lua脚本
    • 了解：分布式锁

  5. 消息队列

    • 消息队列比较
    • 如果处理重复（幂等）
    • 保证数据不丢：事务，ACK，持久化
    • 消息队列作用：解耦，消峰
    • 高可用搭建

  6. 大数据处理（加分）

    • hadoop
    • hbase（重点）
    • spark
    • hive

  7. 虚拟化

    • 熟悉：Docker/DockerCompose

    • 了解：K8S（加分）

  8. Spring

    • Bean加载流程
    • 事务传播
    • AOP实现原理
    • CGLIB
    • SpringMVC处理步骤
