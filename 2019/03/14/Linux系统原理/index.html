Linux系统原理

Posted on 2019-03-14

进程

进程对应Linux中一个程序运行实体（不一定再运行，可以在创建状态，僵尸状态等）及其上下文环境。

Linux系统没有传统意义的线程，对应的是轻量级进程（LWP），与普通进程不同的是，共享其父进程的上下文环境。

内核把进程列表放入任务队列中(task_list)，采用双向循环链表实现，其中的节点为task_struct，在32位集器上，大概由1.7KB。

Linux使用slab分配task_struct结构，这样可以达到对象复用和缓存着色（参见内存管理）。

task_struct存储在进程内核栈（参见进程管理）的尾端，方便寻址。

进程采用PID（进程标识符）来表示，PID默认最大值是32768。

进程状态

  1. TASK_RUNNING：进程可执行，正在执行或在运行队列中
  2. TASK_INTERRUPTIBLE：可中断，进程被阻塞或睡眠，等待事件唤醒（定时器，I/O事件通知等等）
  3. TASK_UNINTERRUPTIBLE：不可中断，接收到信号也不会中断（参见中断）
  4. TASK_TRACED：被其它进程跟中
  5. TASK_STOPED：进程停止运行，没有运行，也不能投入运行。

进程家族树

所有进程都是PID为1的init进程的后代

task_struct通过parent指针和children链表来存储父子关系。

进程操作

创建

Linux进程创建分为两步fork()和exec()

fork拷贝当前进程，并创建一个子进程，与父进程的不同之处只是PID，PPID（父进程ID）和资源统计量不同。

exec：载入程序并执行

fork采用写时复制技术，避免进程复制导致fork时间过长（但要复制页表）

fork执行流程

  1. 创建内核栈，thread_info和task_struct
  2. 确保当前用户拥有进程数没有超过限制
  3. 将状态设置为TASK_UNINTERRUPTABLE
  4. 分配PID
  5. 根据clone（见下面）方法的标记，拷贝对应数据

vfork：不会复制页表，但只有vfork调用返回，之前的进程才能运行

clone

常见flags类型：

  1. CLONE_FILES：共享打开的文件
  2. CLONE_FS：共享文件系统信息
  3. CLONE_SIGHAND：共享被阻断的信号

内核进程

内核进程没有独立的空间，只运行在内核空间中。

创建：

进程终结

  1. 将flag设置为PF_EXITING
  2. 删除内核定时器
  3. 释放占用的mm_struct引用计数
  4. 调用sem __exit()函数，如果排队等候IPC信号，则离开队列
  5. 释放文件系统和文件引用计数
  6. 设置exit_code
  7. 调用exit_notify给父进程发信号
  8. do_exit()调用schedule()切换到新进程

其中，父进程通过wait()来获取子进程的退出通知。

子进程退出不会释放task_struct，仍然会占用内核空间，所以父进程应当在wait方法中，释放子进程占用资源。

否则会产生僵尸进程（进程自己退出会进入僵尸状态），影响系统效率。

进程通信

通信方式：

管道：通过共享文件系统实现

  1. 普通管道：只能单向传输，只能用于父子或兄弟进程
  2. 流管道：半双工，只能用于父子或兄弟进程
  3. 命名管道：半双工，可以在不想管的进程间通信

信号量

一个计数器，防止某进程访问共享资源时，其它进程也访问。

消息队列

用链表组织的队列，存放在内核中，由消息队列标识符标志。相比管道空间更大

信号

通知进程某个事件发生

共享内存

最快的IPC方式，由一个进程创建，但多个内存可以访问

套接字(socket)

可用于不同集器的访问

调度

抢占式和非抢占式：

  1. 抢占：由调度器来决定是否执行，到达条件，进程主动挂起
  2. 非抢占：由进程自己决定是否挂起

Linux是抢占式的

2.6.23的内核实现了公平式调度(CFS)，公平和非公平体现在调度的时间片大小。

nice值

linux使用nice值来标记进程优先级，范围式-20到19。

Unix及老内核实现，nice表示时间片的比例，nice值越大，获得的时间片越小，时间片的分配要给出默认值。该算法时间复杂度为O(1)

问题：

  1. 需要分配一个绝对的时间片，必须式定时器节拍的整数倍（x86是10ms），并且会随着定时器节拍改变。
  2. 由于唤醒机制，进程可以打破公平原则，获取更多的时间片。

CFS

CFS使用一个结构来存储运行统计信息（已经运行了多少时间片），优先运行已经运行时间片少的进程。

此时nice值的作用是作为增加时间片的权重（越大记录的时间片增加越多）。

CFS采用红黑树来记录可运行进程队列。

sechdule()方法从队列中取出一个任务执行。

重要：睡眠的进程会从可执行红黑树移除

调度策略

  1. SCHED_NORMAL：即普通类型调度，优先级比下两种低。下面两种任务执行完，才会进入SCHED_NORMAL
  2. SCHED_FIFO：实时模式，不用时间片，按先入先出原则执行
  3. SCHED_RR：实时模式，指定时间内执行不完就停止执行

抢占

用户抢占：内核返回用户空间时，如果设置了need_resched标记，会导致schedule()被再次调用，选择下一个合适的进程。

内核抢占：2.6后实现，Unix中内核代码必须在执行完成或阻塞时让出时间片。内核抢占可以在执行中让出时间片，只要调度是安全的（没有使用锁）。

系统调用

系统调用用于使用内核提供的应用程序接口，系统调用时处于进程上下文。

系统调用好处：

  1. 对硬件访问提供抽象
  2. 保护系统的稳定和安全，为资源访问提供权限

原理：

  1. 将参数地址和调用号存入寄存器
  2. 通过中断指令，进入内核空间
  3. 执行完毕返回中断点（或进行下一步调度）

与进程空间进行交互

copy_to_user()：向用户空间写数据
copy_from_user()：从用户空间读数据

中断

CPU执行某些指令时时间可能过长（如向硬盘写数据），可以在执行开始后，让出时间片执行其它任务，在这个任务完成后，调用预先设置的中断处理程序。这样可以避免cpu在等待I/O等慢设备操作时，处于空闲状态。

实现方式：

  1. 轮询（可编程中断）：在慢设备中设置一个标记位，CPU通过总线读取标记位，标记位改变触发中断。这种处理会受慢设备标记读取速度影响，因为要轮询，也会消耗额外的CPU时间。
  2. 触发：慢设备指令执行完成后，主动触发CPU。缺点是需要慢设备提供该功能，硬件设计复杂。

分类：

  1. 程序中断：如算术溢出，非法指令等
  2. 时钟中断：处理器计时器触发
  3. I/O中断：I/O控制器产生
  4. 硬件失效中断

优先级：

中断正在执行时触发中断，一般处理是直接忽略该中断信号
另外可以设置优先级，优先级高的可以中断优先级低的正在执行的中断程序

DMA：直接内存访问

访问I/O（读）的通常流程：

  1. CPU检测I/O缓存区是否有数据
  2. 有数据则将数据复制到内存

缺点是：

  1. CPU复制内存会占用CPU时间（相对较长）
  2. 如果CPU长时间未处理可能会导致缓存区满，I/O数据丢失

可以使用额外的DMA控制器，将I/O缓存区的数据复制到内存中，节省CPU消耗。

复制完成，I/O设备通知CPU即可。

上半部和下半部

中断的执行时间应当比较短（影响其它中断处理），但有时候中断又需完成较多的任务。

可以将中断分为上半部（处理最简单的必须任务），下半部，放入待处理队列，后续执行。

API

  • irq为中断号
  • handler为中断处理程序

flags：

  1. IRQF_DISABLED：如果设置，内核处理其它程序时禁止其它所有中断
  2. IRQF_SAMPLE_RANDOM：设备产生的中断对熵池（用于随机数生成）有贡献
  3. IRQF_TIMER：为系统定时器中断准备
  4. IRQF_SHARED：在多个中断处理中共享

释放中断：

中断上下文

中断执行时，内核处于中断上下文。

中断上下文不能睡眠，并由严格时间限制。

中断上下文共享中断进程的内核栈

下半部

实现下半部的方式包括：

  1. 软中断
  2. tasklet
  3. 工作队列

软中断

软中断是编译期分配的，由softirq_action表示

系统调用do_softireq来执行软中断中的程序。

软中断保留给系统中对事件要求最严格以及最重要的下半部使用。

软中断处理程序时，允许响应中断，但不能休眠。

tasklet

tasklet通过软中断实现。

tasklet使用链表存储tasklet_struct，来实现处理队列。

分为高优先级和普通队列。

调度步骤：

  1. 检查tasklet的状态是否为TASKLET_STATE_SCHED，如果是，则已经被调度过，函数返回
  2. 调用_tasklet_schedule()
  3. 保存中断状态，禁用本地中断
  4. 把需要调度的tasklet加到每个处理器一个的tasklet_vec链表或tasklet_hi_vec链表（高优先级）
  5. 唤醒TASKLET_SOFTIRQ或HI_SOFTIRQ软中断，这样在下一次调用do_softirq()时就会执行该tasklet
  6. 恢复状态并返回

工作队列

工作队列的上下文是进程上下文。

如果推后的任务需要休眠使用工作队列，否则使用tasklet

会由一个线程执行工作队列里的内容

  1. 如果队列时空的，则休眠
  2. 不是空的，则执行

内核同步

常见概念：

SMP：对称CPU处理器，即多个独立的CPU。和多核的区别是无法对缓存和异步做硬件级协调，需要操作系统完成
多核：一个处理器，多个核心，可以并行操作

乱序指令

  1. 流水线计数，CPU可以将互不干扰，不影响前后多级的指令同时执行
  2. 指令重排，重新排序指令，最大化流水线并行

乱序指令重排不会影响单个CPU的指令执行，但可能影响SMP或多核应用程序执行。

原子操作

期望是a=a+2

但实际：

  1. CPU1将a的值存入寄存器
  2. CPU2将a的值存入寄存器
  3. CPU1执行加操作，此时a=a+1
  4. CPU2执行加操作，此时a=a+1
    最终a只加了一次，这是因为该操作不是原子操作，出现了脏读

使用加锁，或CAS指令可以解决

局部性原理

  1. 空间局部：如果一个变量被访问，那么他临近的变量更可能被访问
  2. 时间局部：如果一个变量被访问，那么再次访问该变量的机会会增大
  3. 顺序局部：指令倾向于顺序执行

CPU缓存

CPU缓存距离CPU更近，不需要通过总线访问。CPU的速度比内存快很多。

缓存的执行流程：

读：

  1. 查看缓存是否在CPU缓存中（内存通过hash映射到缓存中），如果在，则返回
  2. 不在将内存加载到缓存行（缓存单元，缓存及邻近字），并返回

写（什么时候写入内存由内存策略决定）：

  1. 查看该CPU缓存中是否有数据，如果有且不是送入字节的内存区域，采用LRU算法决定是否置换
  2. 如果失败，写入内存

假设缓存命中率是95%，访问内存100us，访问缓存5us，那么不使用缓存100us，使用缓存(5us+0.25us)+4.75=10us，其中0.25us是访问内存时，也要先访问缓存。

可以设计多级缓存，一般为两级，一级比另一级大

CPU缓存在不同的核之间是不共享的，多核执行可能会出现不一致的情况，这时候需要强制重载或写入内存

内核同步方法

自旋锁

操作流程通常为：

自旋锁相对于阻塞锁，会跑完时间片，占用额外的CPU时间，但响应会更快，适合加锁时间短的操作。

自旋锁可以设置读写锁

读写锁在写锁被占用时，读锁和写锁都不能获取；读锁被占用时，写锁不能获取，读锁可以获取

信号量

信号量提供一定数量的临界资源，在临界资源被消耗光时，再尝试获取，将把进程设置为TASK_INTERRUPTIBLE状态。

在重新存在资源时唤醒。

信号量可以支持读写

信号量可以使用trylock自旋

Mutex

Mutex相当于上限为1的信号量。

内存屏障

rmb()：创建读内存屏障，跨域rmb()的读操作不会发生重排序

wmb()：写操作不会发生重排序

mb()：rmb和wmb的组合

定时器和时间管理

节拍率：每秒触发时钟中断的次数，x86架构是100

节拍率高的优势：

  1. 内核定时器更准确
  2. 提高抢占调度的准确性
  3. 资源统计和运行时间统计解析读更高

劣势：

  1. 会频繁打乱高速缓存，并增加耗电

内存管理

内核把物理页作为内存管理的基本单位，32位系统支持4KB的页，64位系统支持8KB的页。

页的结构：

区

由于硬件限制，内核并不对多有的页一视同仁

分类：

  1. ZONE_DMA：包含的页能用来DMA操作
  2. ZONE_DMA32： 只能被32位设备访问
  3. ZONE_NORMAL： 能够正常映射的页
  4. ZONE_HIGNEM：高端内存，不能永久映射到内核空间，x86-64没有高端内存

API

kmalloc

可以获得一块内核内存，分配的内存时物理连续的。

vmalloc

SLAB

SLAB由一个或多个物理上连续的页组成，slab有三种状态，满，部分满，空

slab使用链表存储，使用缓存着色和对象缓存，防止反复申请和释放页。

缓存着色：防止多个对象映射到相同的高速缓存行，使用的是CPU缓存的选路技术

在栈上静态分配内存

用户栈可以预先分配内存，也能动态增加。

内核栈只有4KB或8KB。

高端内存映射

高端内存的页不能永久映射在地址空间上。

使用高端内存需要创建临时页。

页表

页表是物理内存和逻辑页的映射关系，每个进程都有自己的页表。

二级页表：提供目录项，目录指向后续页表。使用页表目录+偏移来表示具体哪个页。只存储使用的目录。

虚拟文件系统VFS

VFS提供标准的文件系统访问接口。

VFS对象：

  1. 超级块节点：提供已安装的文件系统
  2. 索引节点对象：代表一个具体的文件
  3. 目录项对象：代表一个目录项
  4. 文件对象：代表进程打开的文件

目录项缓存

遍历路径名中所有元素并将它们逐个解析成目录对象，时间城北，太大。内核将目录项对象放入缓存中。

缓存包含：

  1. 被使用的目录项链表
  2. 最近被使用的双向链表
  3. 散列和响应的散列函数快速将给定路径解析为相关目录对象

块I/O

块设备：可以随机访问固定大小的设备
字符流设备：只能顺序访问的设备

缓冲区：一个块被调入内存，会先进入缓冲区，包含一个或多个扇区，但不会超过一页

请求队列：块设备将挂起的请求放入队列中

调度

以硬盘为例，每次寻址时间过长，如果以内核请求次序访问，性能会很差。

提交前会先进行合并与排序，负责提交的子系统叫做I/O调度程序。

算法：

Linus电梯

  1. 有相邻的磁盘操作，则合并
  2. 有驻留时间过长的请求，则把新请求放入队尾，防止饥饿
  3. 以扇区方向为次序排序

进程地址空间

进程地址空间由可寻址的虚拟内存组成。

进程只能访问有效的内存区域内的空间，内存区域有可读，可写，可执行属性

内存包含各种内存对象，比如：

  • 可执行文件代码的内存映射
  • 可执行文件的已初始化全局变量的内存映射
  • 进程用户空间栈
  • 动态链接库的共享代码段
  • 内存映射文件
  • 共享内存段
  • 匿名内存映射，如malloc()

内存描述符

内存描述符包含可用内存区域链表等等。

以线程方式clone的对象，不包含单独的内存描述符。

VMA

虚拟内存的一些标志

如：

  • VM_READ/VM_WRITE/VW_EXEC/VM_SHARED：可读写执行共享标记
  • VM_GROW/VM_DOWN：可增加
  • VM_IO：区域映射I/O空间

mmap

mmap()和do_mmap用来创建线性地址空间，并映射到页上，用于文件和内存的共享（子进程）。

mummap()和do_mummap：删除地址区间

页高速缓存

写缓存：程序执行写时先写到缓存，并通过一定策略写入磁盘

读缓存：读取文件时，先从页上查找

缓存回收：

  • LRU
  • 双链策略：维护一个活跃链表和一个不活跃的，非活跃的链表可以被换出，又叫做LR/2

flusher线程

写入磁盘的时机

  1. 空间内存低于一定阈值，写入磁盘，并回收页
  2. 脏页驻留时间操作一定阈值
  3. sync()和fsync()调用时
